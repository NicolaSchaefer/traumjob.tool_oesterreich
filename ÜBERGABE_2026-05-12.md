## Was diese Session gemacht hat

### Bug gefixt ✅
**Fehler:** `Invalid format specifier ' 1, "7": 3, ...' for object of type 'str'`  
**Ursache:** In `ai.py`, Zeile 336, war `{"6": 1, "7": 3, ...}` unescaped in einem f-String. Python interpretierte `"6"` als Format-Feldname und ` 1, "7": 3, ...` als Format-Specifier.  
**Fix:** `{"6": 1, "7": 3, ...}` → `{{"6": 1, "7": 3, ...}}` (doppelte geschweifte Klammern = Literal in f-Strings).

**Zweiter Fix (gleichzeitig):** In `main.py` → `energie_get` wurde `saved` von String-Keys auf Integer-Keys umgebaut, damit `saved[h]` im Template direkt funktioniert (statt `saved.get(h|string, 0)`).
